# Oil DGA App

این نسخه از پروژه علاوه بر تحلیل DGA، ورود کاربر و ذخیره سوابق تست‌ها را هم پشتیبانی می‌کند.

## چه چیزی اضافه شده

- ثبت‌نام با `شماره موبایل + نام + نام خانوادگی + شرکت + رمز عبور`
- ورود با `شماره موبایل + رمز عبور`
- ذخیره هر تحلیل زیر همان حساب کاربری
- صفحه `سوابق تست‌ها`
- راه‌اندازی سازگار با `GitHub Pages` و بدون بک‌اند اختصاصی

## فایل‌های مهم

- `auth.html`
  - صفحه ورود و ساخت حساب
- `history.html`
  - صفحه سوابق تست‌های هر کاربر
- `auth.js`
  - منطق احراز هویت، سشن، خروج، ثبت پروفایل و خواندن سوابق
- `supabase-config.js`
  - محل قرار دادن `Supabase URL` و `Anon Key`
- `supabase-schema.sql`
  - اسکریپت ساخت جداول و Policyهای امنیتی
- `app.js`
  - منطق فرم، نتیجه تحلیل و ثبت خودکار هر تست

## معماری این نسخه

چون پروژه روی `GitHub Pages` بالا می‌آید و بک‌اند خصوصی ندارد، ورود با موبایل و رمز به این شکل پیاده شده است:

- کاربر در UI شماره موبایل وارد می‌کند
- در لایه `Supabase Auth` این شماره به یک ایمیل داخلی تبدیل می‌شود
- از دید کاربر همچنان تجربه `موبایل + رمز` باقی می‌ماند

نکته مهم:

- برای اینکه این روش کار کند باید `email confirmation` در تنظیمات `Supabase Auth` خاموش باشد
- این روش برای MVP مناسب است
- اگر بعداً تایید واقعی مالک شماره را خواستی، باید `OTP / SMS` اضافه شود

## راه‌اندازی Supabase

### 1. ساخت پروژه

در [Supabase](https://supabase.com/) یک پروژه جدید بساز.

### 2. اجرای SQL

فایل [supabase-schema.sql](/Users/arash/Documents/My Arash/oil-dga-app/oil-dga-app/supabase-schema.sql) را در SQL Editor اجرا کن.

این فایل این‌ها را می‌سازد:

- جدول `profiles`
- جدول `analyses`
- `RLS policies` برای اینکه هر کاربر فقط اطلاعات خودش را ببیند

### 3. تنظیم Auth

در بخش `Authentication` این تنظیم‌ها را انجام بده:

- `Confirm email`: خاموش
- `Site URL`: آدرس GitHub Pages خودت
- `Redirect URLs`: اگر لازم بود همان دامنه GitHub Pages را اضافه کن

مثال:

```text
https://YOUR-USERNAME.github.io
https://YOUR-USERNAME.github.io/YOUR-REPO
```

اگر پروژه در روت دامین گیت‌هاب‌پیجز نیست و زیرنام مخزن بالا می‌آید، حتماً همان مسیر کامل را هم اضافه کن.

### 4. قرار دادن کلیدها

فایل [supabase-config.js](/Users/arash/Documents/My Arash/oil-dga-app/oil-dga-app/supabase-config.js) را باز کن و این دو مقدار را پر کن:

```js
window.SUPABASE_CONFIG = {
  url: "https://YOUR_PROJECT.supabase.co",
  anonKey: "YOUR_SUPABASE_ANON_KEY"
};
```

نکته:

- `anon key` در فرانت‌اند قابل استفاده است و محرمانه‌ی سروری نیست
- `service role key` را هیچ‌وقت داخل این پروژه نگذار

## اجرا

### لوکال

با هر سرور استاتیک ساده می‌توانی پروژه را اجرا کنی.

### GitHub Pages

فایل‌ها را روی همان ریپوی فعلی push کن و GitHub Pages را مثل قبل نگه دار.

برای این نسخه، سرویس خارجی لازم این است:

- `Supabase`

سرویس دیگری لازم نیست، چون فعلاً `SMS / OTP` را وارد نکردیم.

## جریان کاربر

1. کاربر وارد `auth.html` می‌شود
2. حساب می‌سازد یا وارد می‌شود
3. وارد `index.html` می‌شود
4. مقادیر گازها را ثبت می‌کند
5. نتیجه در `results.html` نمایش داده می‌شود
6. همان تست در جدول `analyses` ذخیره می‌شود
7. کاربر سوابقش را در `history.html` می‌بیند
